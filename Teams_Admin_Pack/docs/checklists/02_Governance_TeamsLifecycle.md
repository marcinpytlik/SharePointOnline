# Checklist – Governance i cykl życia Teamów

## A. Tworzenie
- [ ] Konwencja nazw + opis Teamu
- [ ] Minimum 2 ownerów
- [ ] Typ (public/private/hidden membership) zgodny z klasyfikacją danych
- [ ] Domyślne polityki meeting/messaging/app przypisane

## B. Utrzymanie (cyklicznie)
- [ ] Raport: Teamy z < 2 ownerami
- [ ] Raport: Teamy nieaktywne / do archiwizacji
- [ ] Przegląd gości: sponsor, cel, data wygaśnięcia
- [ ] Kanały private/shared: czy są uzasadnione

## C. Archiwizacja
- [ ] Zgoda ownera
- [ ] Komunikat do użytkowników
- [ ] Archiwizacja + ewentualne ograniczenie zmian
- [ ] Retencja zgodna z Purview (jeśli używane)

## D. Usunięcie
- [ ] Potwierdzenie biznesowe
- [ ] Plan odzysku (jeśli wymagany)
- [ ] Dokumentacja decyzji
