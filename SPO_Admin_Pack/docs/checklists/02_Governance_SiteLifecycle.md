# Checklist – Governance i cykl życia site’ów

## A. Tworzenie (Provisioning)
- [ ] Konwencja nazw (dział/projekt)
- [ ] Minimum 2 właścicieli (Owners)
- [ ] Ustalony typ: Team site / Communication site
- [ ] Ustalona polityka sharing per-site
- [ ] Wersjonowanie i limity wersji w bibliotekach

## B. Utrzymanie
- [ ] Miesięczny/kwartalny raport: ownerzy, sharing, storage, goście
- [ ] Sprzątanie „orphaned sites” (bez ownera)
- [ ] Audyt unikalnych uprawnień (biblioteki/itemy)

## C. Zamknięcie / archiwum
- [ ] Decyzja ownera o archiwizacji
- [ ] Ograniczenie edycji / zmiana uprawnień na read-only
- [ ] Retencja zgodna z polityką (Purview)
- [ ] Usunięcie po czasie + procedura przywracania
